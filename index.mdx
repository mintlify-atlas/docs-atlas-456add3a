OpenSSF Scorecard is an automated security tool that evaluates open source projects against a set of security best practices — called "checks" — and assigns each a score from 0 to 10. Use Scorecard to harden your own project's security posture, or to evaluate the risk of dependencies before you adopt them.

## What Scorecard checks

Scorecard evaluates your repository across multiple security dimensions. Each check returns a score of 0–10 and a risk level, feeding into a weighted aggregate score.

<CardGroup cols={3}>
  <Card title="Branch protection" icon="code-branch" href="/checks/reference">
    Verifies that default and release branches are protected against force pushes and unreviewed merges.
  </Card>
  <Card title="Code review" icon="magnifying-glass-code" href="/checks/reference">
    Checks whether changes require review before being merged into protected branches.
  </Card>
  <Card title="Dependency management" icon="box-open" href="/checks/reference">
    Detects pinned dependencies, active update tools, and known vulnerabilities.
  </Card>
  <Card title="Signed releases" icon="certificate" href="/checks/reference">
    Confirms that published releases are cryptographically signed for supply chain integrity.
  </Card>
  <Card title="Token permissions" icon="key" href="/checks/reference">
    Ensures GitHub workflow tokens follow the principle of least privilege.
  </Card>
  <Card title="Fuzzing & SAST" icon="bug" href="/checks/reference">
    Detects use of fuzz testing tools and static analysis in CI pipelines.
  </Card>
</CardGroup>

## Three ways to use Scorecard

<Tabs>
  <Tab title="CLI">
    Run a one-off scan on any public repository with a single command:

    ```bash
    export GITHUB_AUTH_TOKEN=<your-token>
    scorecard --repo=github.com/ossf/scorecard
    ```

    See [CLI usage](/usage/cli) for all flags and output formats.
  </Tab>
  <Tab title="GitHub Action">
    Add the Scorecard GitHub Action to your repository for continuous monitoring. Results appear in the Security tab as code scanning alerts.

    See [GitHub Action setup](/usage/github-action) for installation instructions.
  </Tab>
  <Tab title="REST API">
    Query pre-calculated scores for over one million open source projects without running the tool yourself:

    ```bash
    curl https://api.scorecard.dev/projects/github.com/ossf/scorecard
    ```

    See [REST API](/usage/rest-api) for full endpoint documentation.
  </Tab>
</Tabs>

## Get started

<Steps>
  <Step title="Install Scorecard">
    Download the binary, use Docker, or install via Homebrew. See [Installation](/installation).
  </Step>
  <Step title="Authenticate">
    Create a GitHub personal access token and set it as `GITHUB_AUTH_TOKEN`. See [Authentication](/authentication).
  </Step>
  <Step title="Run your first scan">
    Run `scorecard --repo=<your-repo>` and review the results. See [Quickstart](/quickstart).

  <Step title="Understand and improve your score">
    Read the [Checks reference](/checks/reference) and [Beginner's guide](/checks/beginner-guide) to prioritize improvements.

</Steps>
