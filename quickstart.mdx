---
title: "Get started with OpenSSF Scorecard"
description: "Install the Scorecard CLI, authenticate with GitHub, run your first scan, and learn how to interpret the results in under ten minutes."
---

This guide walks you through installing the Scorecard CLI, authenticating with GitHub, running a scan on a public repository, and reading the output. By the end you will have a working Scorecard setup and understand what the results mean.

If you prefer a no-install option, the [GitHub Action](/usage/github-action) runs Scorecard automatically on every push and surfaces results as security alerts in your repository.

<Steps>
  <Step title="Install Scorecard">
    The quickest way to install Scorecard is with Homebrew on macOS or Linux:

    ```bash
    brew install scorecard
    ```

    For Docker, standalone binary, Nix, or AUR installation, see the [installation guide](/installation).

    Verify the install:

    ```bash
    scorecard version
    ```
  </Step>

  <Step title="Authenticate with GitHub">
    GitHub enforces strict rate limits on unauthenticated API requests. You must set an auth token before running a scan.

    Create a [classic GitHub personal access token](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/creating-a-personal-access-token#creating-a-personal-access-token-classic) with the `public_repo` scope, then export it:

    <CodeGroup>

    ```bash Linux / macOS
    export GITHUB_AUTH_TOKEN=<your-token>
    ```

    ```bat Windows
    set GITHUB_AUTH_TOKEN=<your-token>
    ```

    </CodeGroup>

    For GitLab repositories or advanced options like GitHub App credentials, see the [authentication guide](/authentication).
  </Step>

  <Step title="Run your first scan">
    Pass any public repository URL to `--repo`:

    ```bash
    scorecard --repo=github.com/ossf-tests/scorecard-check-branch-protection-e2e
    ```

    Scorecard runs all checks in parallel and prints progress to stderr. When complete, it prints the results table:

    ```
    RESULTS
    -------
    Aggregate score: 7.9 / 10

    Check scores:
    |---------|------------------------|--------------------------------|---------------------------------------------------------------------------|
    |  SCORE  |          NAME          |             REASON             |                         DOCUMENTATION/REMEDIATION                         |
    |---------|------------------------|--------------------------------|---------------------------------------------------------------------------|
    | 10 / 10 | Binary-Artifacts       | no binaries found in the repo  | github.com/ossf/scorecard/blob/main/docs/checks.md#binary-artifacts       |
    |---------|------------------------|--------------------------------|---------------------------------------------------------------------------|
    | 9 / 10  | Branch-Protection      | branch protection is not       | github.com/ossf/scorecard/blob/main/docs/checks.md#branch-protection      |
    |         |                        | maximal on development and all |                                                                           |
    |         |                        | release branches               |                                                                           |
    |---------|------------------------|--------------------------------|---------------------------------------------------------------------------|
    | ?       | CI-Tests               | no pull request found          | github.com/ossf/scorecard/blob/main/docs/checks.md#ci-tests               |
    |---------|------------------------|--------------------------------|---------------------------------------------------------------------------|
    | 0 / 10  | CII-Best-Practices     | no badge found                 | github.com/ossf/scorecard/blob/main/docs/checks.md#cii-best-practices     |
    |---------|------------------------|--------------------------------|---------------------------------------------------------------------------|
    | 10 / 10 | Code-Review            | branch protection for default  | github.com/ossf/scorecard/blob/main/docs/checks.md#code-review            |
    |         |                        | branch is enabled              |                                                                           |
    |---------|------------------------|--------------------------------|---------------------------------------------------------------------------|
    | 10 / 10 | Token-Permissions      | tokens are read-only in GitHub | github.com/ossf/scorecard/blob/main/docs/checks.md#token-permissions      |
    |         |                        | workflows                      |                                                                           |
    |---------|------------------------|--------------------------------|---------------------------------------------------------------------------|
    | 10 / 10 | Vulnerabilities        | no vulnerabilities detected    | github.com/ossf/scorecard/blob/main/docs/checks.md#vulnerabilities        |
    |---------|------------------------|--------------------------------|---------------------------------------------------------------------------|
    ```

    To scan your own repository, replace the `--repo` value with your repo's URL.
  </Step>

  <Step title="Interpret the results">
    Each row in the results table shows:

    - **Score** — a value from 0–10, or `?` if the check could not run (for example, no pull requests were found for `CI-Tests`).
    - **Name** — the check that ran. See the [checks reference](/checks/reference) for a description of each.
    - **Reason** — a short explanation of why the score was given.
    - **Documentation/Remediation** — a link to the detailed check documentation and remediation steps.

    The **aggregate score** is a risk-weighted average across all checks. A score of `?` does not count against the aggregate.

    For more detail on why a check produced a specific score, add `--show-details`:

    ```bash
    scorecard --repo=github.com/ossf-tests/scorecard-check-branch-protection-e2e \
      --checks=Branch-Protection \
      --show-details
    ```

    This adds a **Details** column to the output with per-rule findings — for example, which branch protection settings are enabled or missing on each branch.

    <Tip>
      Use `--checks=<name1>,<name2>` to run only specific checks while iterating on fixes. See [CLI usage](/usage/cli) for all available flags.
    </Tip>
  </Step>
</Steps>
