---
title: "Authenticate Scorecard with GitHub and GitLab"
description: "Set up GitHub personal access tokens, GitHub App credentials, or a GitLab token so Scorecard can scan repositories without hitting API rate limits."
---

Scorecard queries the GitHub and GitLab APIs to run its checks. Both platforms enforce strict rate limits on unauthenticated requests — without a token, most scans will fail or return incomplete results. This page explains how to set up credentials for each platform.

## GitHub authentication

### Personal access token

The simplest way to authenticate is with a [classic GitHub personal access token](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/creating-a-personal-access-token#creating-a-personal-access-token-classic). When creating the token, select the `public_repo` scope.

Set the token in one of the following environment variables — Scorecard accepts any of them:

- `GITHUB_AUTH_TOKEN`
- `GITHUB_TOKEN`
- `GH_AUTH_TOKEN`
- `GH_TOKEN`

<CodeGroup>

```bash Linux / macOS
export GITHUB_AUTH_TOKEN=<your-token>
```

```bat Windows
set GITHUB_AUTH_TOKEN=<your-token>
```

</CodeGroup>

You can provide multiple tokens separated by commas. Scorecard rotates through them in round-robin order to distribute API usage:

<CodeGroup>

```bash Linux / macOS
export GITHUB_AUTH_TOKEN=<token1>,<token2>,<token3>
```

```bat Windows
set GITHUB_AUTH_TOKEN=<token1>,<token2>,<token3>
```

</CodeGroup>

### GitHub App

For higher rate-limit quotas, use a [GitHub App Installation](https://docs.github.com/en/developers/apps/building-github-apps/creating-a-github-app). If you have a GitHub App with an installed key file, set these three environment variables:

<CodeGroup>

```bash Linux / macOS
export GITHUB_APP_KEY_PATH=<path-to-key-file>
export GITHUB_APP_INSTALLATION_ID=<installation-id>
export GITHUB_APP_ID=<app-id>
```

```bat Windows
set GITHUB_APP_KEY_PATH=<path-to-key-file>
set GITHUB_APP_INSTALLATION_ID=<installation-id>
set GITHUB_APP_ID=<app-id>
```

</CodeGroup>

You can find these values on the GitHub [developer settings](https://github.com/settings/apps) page.

### GitHub Enterprise Server

To scan repositories on a GitHub Enterprise Server instance, set the `GH_HOST` environment variable to your enterprise hostname (without the `https://` prefix):

<CodeGroup>

```bash Linux / macOS
export GH_HOST=github.corp.com
export GITHUB_AUTH_TOKEN=<your-token>

scorecard --repo=github.corp.com/org/repo
```

```bat Windows
set GH_HOST=github.corp.com
set GITHUB_AUTH_TOKEN=<your-token>

scorecard --repo=github.corp.com/org/repo
```

</CodeGroup>

## GitLab authentication

To scan a GitLab repository, create a [GitLab personal access token](https://gitlab.com/-/profile/personal_access_tokens) with the following scopes:

- `read_api`
- `read_user`
- `read_repository`

Then set the token in the `GITLAB_AUTH_TOKEN` environment variable:

<CodeGroup>

```bash Linux / macOS
export GITLAB_AUTH_TOKEN=<your-gitlab-token>

scorecard --repo=gitlab.com/<org>/<project>
```

```bat Windows
set GITLAB_AUTH_TOKEN=<your-gitlab-token>

scorecard --repo=gitlab.com/<org>/<project>
```

</CodeGroup>

### Self-hosted GitLab

Scorecard supports self-hosted GitLab installations. If your instance is at a subdomain (for example, `gitlab.corp.com`), Scorecard detects it automatically. If your instance is served from a path slug (for example, `corp.com/gitlab`), set the `GL_HOST` environment variable:

<CodeGroup>

```bash Linux / macOS
export GITLAB_AUTH_TOKEN=<your-gitlab-token>
export GL_HOST=corp.com/gitlab

scorecard --repo=corp.com/gitlab/<org>/<project>
```

```bat Windows
set GITLAB_AUTH_TOKEN=<your-gitlab-token>
set GL_HOST=corp.com/gitlab

scorecard --repo=corp.com/gitlab/<org>/<project>
```

</CodeGroup>

<Note>
  For an example of using Scorecard in a GitLab CI/CD pipeline, see the [ossf-test/scorecard-pipeline-example](https://gitlab.com/ossf-test/scorecard-pipeline-example) project on GitLab.
</Note>
